# Applications

## `weread-qt`

This is the current product application. It is a Qt 6 / Qt Quick program built with the official reMarkable `chiappa` SDK.

Start here for UI, reader, account, comments, stylus, network, frontlight and power work.

## `weread-move`

This directory has two roles:

1. `lib/` and `tools/` are the Lua helper layer used by the Qt app for WeRead networking, chapter/content handling and cache operations.
2. `app.lua`, `native_app.lua`, `views/`, `rm-weread.sh` and `external.manifest.json` are legacy prototypes.

Do not deploy the legacy prototype when validating the current app. The current install entry is generated by `scripts/install-weread-qt-appload.sh` and launches `/home/root/weread-qt/rm_weread_qt` through `scripts/weread-qt-session.sh`.
