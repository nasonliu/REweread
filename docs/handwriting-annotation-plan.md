# 手写批注与云端 OCR 方案

## 已实现的第一阶段

- 手写工具胶囊统一提供四种颜色、画线、自由手写、橡皮和“笔记显示”开关。
- 落笔预览使用设备帧缓冲的单色极速局部刷新；停笔后再合并为所选颜色并由页面图层完成一次稳定刷新，避免每个采样点触发整条 Qt 渲染链。
- 自由手写保存为原始多笔划，不以本地 OCR 替代笔迹。
- 保存时以笔迹到正文行框的距离判断归属：靠近正文时锚定完整段落（文本范围、摘录和上下文）；远离正文时保留为本页自由笔记，并保存归一化页内位置。
- 重排后，段落笔记跟随其正文段落显示在右侧笔记栏；无法可靠关联的自由笔记保留在原页的相对空白位置。
- 点手写笔记可打开识别面板。只有用户点击“识别文字”才会把临时 PNG 经 HTTPS 发送给百度；候选文本须由用户点击确认，才会写入该笔记。原始笔迹不会被删除。
- 书城搜索可用本地拼音输入法，也可以用同一手写识别面板确认文字后填入搜索框。

## 数据边界

`reader-paragraph-notes.json` 只保存笔划、锚点、显示颜色、可选的用户确认 OCR 文本和时间；不保存百度 access token，也不把书籍正文、整页截图或笔记上传作训练数据。OCR 凭据单独保存在 owner-only 的 `baidu-ocr.json`。

## 后续增量

1. 在真机以正文页验证段落锚定、页内自由笔记和字体/边距重排。
2. 为已识别笔记提供“重新识别”和“移除文本、只保留笔迹”。
3. 在阅读器内补充文字搜索入口：拼音输入或手写 OCR 确认后定位到正文结果；画线仍以真实渲染文字范围为准，不用 OCR 猜测正文范围。
4. 仅在用户授权真机测试后，验证浏览器证书提示、配对码过期、Wi-Fi 切换和百度错误码的无敏感信息呈现。
