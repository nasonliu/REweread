# REweread 1.5.0

## Source-only 1.5.0 milestone

REweread 1.5.0 brings the current Qt reader's Chinese input and handwriting
work into one documented source milestone. It is an unofficial,
noncommercial, source-only update. This change does not publish a tag, binary
or public GitHub Release.

## Highlights

- Pinyin, handwriting and English input modes share one on-screen keyboard.
- Pinyin and OCR candidates have explicit previous/next page controls.
- The reader pen capsule separates color from tool choice and supports
  text-snapped highlighting, free handwriting, erasing, OCR, note visibility
  and confirmed page clearing.
- Free ink stays at the page location where it was written. Paragraph-linked
  notes and page-free notes can retain their original strokes without OCR.
- A selected nearby-stroke block can be sent to Baidu handwriting OCR and the
  recognized text is attached inline; pure marks and underlines need not be
  recognized.
- Direct framebuffer ink, palm rejection and delayed settled refresh improve
  pen latency on Paper Pro Move.
- The My page can start a short-lived HTTPS pairing service so the user enters
  Baidu credentials from a phone or computer instead of placing secrets in the
  repository or shell history.

## Upgrade

Agents must follow [the 1.5 upgrade guide](../agent-upgrade-v1.5.md). Existing
users keep `/home/root/.local/share/rm-weread/`; this includes account state,
progress, annotations and any existing OCR configuration. The atomic installer
stages the new application and retains `.previous` rollback directories.

Human users should read the [quick application guide](../quick-start-user-guide.md)
and the [Baidu OCR setup guide](../baidu-ocr-configuration-flow.md).

## Validation completed for this source milestone

- Repository safety scan.
- Complete static validator suite, including release metadata, OCR transport,
  pinyin, free-note anchoring and fast-ink contracts.
- Shell syntax validation.
- Official chiappa SDK cross-build.
- Live-device launch with direct ink ready and no QML, framebuffer or input
  overrun errors during the handwriting-fix iterations.

These checks do not remove the clean-device install/uninstall, long-duration
battery, service authorization or upstream rights blockers in
`docs/release-checklist.md`.

## Distribution boundary

Do not attach an application binary, SDK, fonts, XOVI, AppLoad, KOReader,
`weread.koplugin`, credentials, user data, covers, comments or books to a
release. The repository remains under PolyForm Noncommercial 1.0.0 and is not
officially authorized by Tencent, WeRead or reMarkable.
