# REweread 2.0.0

## Source-only 2.0.0 milestone

2.0.0 completes the handwriting and Magic Notebook work as a source-only,
unofficial, noncommercial milestone. It does not create a binary release,
tag, or public GitHub Release.

## What changed

- Magic Notebook clears a handwritten question before its AI reply begins.
- Replies are rendered as native font-to-skeleton pen paths with small direct
  framebuffer updates, then dissolve locally instead of flashing the page.
- Baidu OCR and DeepSeek remain separately configured optional cloud services.
- Named persona cards define a character's name, era, history and speaking
  style so identity questions have a consistent answer.

## Configuration and privacy

Users create their own Baidu OCR and DeepSeek credentials and enter them only
through **我的 → 百度 OCR → 开启浏览器配置**. See
[the configuration guide](../baidu-ocr-configuration-flow.md) and
[the user guide](../quick-start-user-guide.md). Existing credentials survive
an application upgrade; entering one service never clears the other.

## Upgrade and distribution boundary

Agents must read [the 2.0 upgrade guide](../agent-upgrade-v2.0.md). Do not
commit or distribute binaries, SDKs, fonts, API keys, user data, book content,
XOVI, AppLoad, KOReader or `weread.koplugin`.
