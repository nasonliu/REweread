# REweread 1.0.0-rc.1

This release candidate freezes the first complete Qt product milestone for reMarkable Paper Pro Move.

## What is ready

- Nine-cover shelf, book detail, chapter-first opening and whole-book download.
- Chinese reader typography and persistent reading settings.
- Table of contents, progress, images, captions, notes and cached social comments.
- Stylus highlights with touch rejection.
- QR login, Wi-Fi, frontlight, battery, power button, magnetic cover and stock-system return.
- Official chiappa SDK build and repository validation.

## Important limitations

- Source-only release candidate: no installable binary is attached.
- Unofficial and noncommercial; no Tencent, WeRead or reMarkable authorization is implied.
- The content/network helper still depends on user-installed KOReader LuaJIT and `weread.koplugin`.
- WeRead Web endpoints are unstable and account/service behavior may change.
- Only the compatibility matrix baseline has been tested.
- Clean-device installation, uninstall and long-duration battery measurement are not yet complete.

Validation evidence and remaining gates are listed in `docs/releases/v1.0.0-rc.1-validation.md`.

## Installation

This release is intended for AI-assisted source builds. Read `AGENTS.md`, `README.md`, `docs/compatibility.md` and `docs/legal-and-commercial-use.md` before touching a device. Developer Mode must be enabled by the device owner after backup and factory-reset confirmation.

## Distribution boundary

The source archive intentionally excludes the app binary, SDK, fonts, XOVI, AppLoad, KOReader, `weread.koplugin`, credentials, caches and book content. Do not attach unofficial dependency bundles to this release.
