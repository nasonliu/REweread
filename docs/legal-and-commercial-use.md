# 法律、许可与商业使用说明

本文是项目维护与风险识别说明，不构成针对任何个人、公司或司法辖区的法律意见。准备公开分发、收费服务或商业部署前，应由熟悉软件许可、著作权、平台服务条款和个人信息保护的律师审查。

## 结论

1. 本仓库当前及后续版本采用 `PolyForm Noncommercial License 1.0.0`，只许可非商用目的。
2. 项目应称为“源码可用”，不应称为 OSI 意义上的“开源软件”。OSI 的开源定义要求允许商业使用。
3. 非商用许可证只能限制本仓库权利人拥有的代码，不能授予微信读书服务、书籍内容、评论、账号数据、商标、第三方插件或设备运行时的权利。
4. 微信读书用户协议对未经腾讯授权的第三方接入、插件、工具和传播有明确限制，其中部分限制不以是否商业使用为条件。
5. 因此，“免费、个人研究、没有盈利”并不自动代表项目符合微信读书协议或其他适用法律。

## 哪些行为属于本项目明确禁止的商业使用

在当前许可证下，不得将本项目用于商业目的，包括但不限于：

- 出售应用、修改版、安装包或下载权限。
- 收费安装、代刷机、维护、技术支持或账号服务。
- 把应用预装到设备后销售、出租或作为赠品促销。
- 提供付费订阅、托管接口、云服务或企业内部业务工具。
- 通过广告、推广佣金、数据服务或其他商业模式变现。
- 代表公司、个体经营者或其他商业组织部署到经营活动中。

许可证原文是判断具体用途是否被允许的最终依据。用途不明确时，应停止使用并咨询权利人和律师。

## 微信读书相关风险

项目目前使用用户绑定的 Skill API、登录 Cookie 和未公开 Web 接口读取书架、章节、图片、进度、评论等数据。这些接口不是本项目拥有的公开 SDK。

[微信读书用户协议](https://cdn.weread.qq.com/app/weread_user_agreement_android_s.html)第 6.1.2 节包含对以下行为的限制：

- 未经授权的第三方软件、插件、外挂或系统登录、使用服务。
- 获取或改变客户端与服务器的交互数据。
- 制作、运营或向公众传播相关工具，即使不以商业为目的。
- 使用、复制、链接、转载、汇编或发布腾讯拥有知识产权的内容。

项目 README 中的“非官方”“非商用”和免责声明不能替代腾讯的书面许可，也不能对抗书籍作者、出版社、评论作者或其他权利人的权利。

## 历史 MIT 版本

本项目早期公开历史使用 MIT 许可证。MIT 明确允许商业使用。许可证变更可以约束之后发布的新版本，但通常不能撤销他人已经对旧版本合法取得的许可。

因此：

- 不要声称旧 MIT 副本已经自动变成非商用。
- 删除旧提交、强制推送或删除仓库不能可靠撤销已经授出的许可。
- 新功能、修复和发布应只在当前非商用许可证下提供。
- 若旧版商业利用成为现实问题，应由律师根据具体副本、时间、贡献和司法辖区处理。

## 如果未来确实要商业化

商业化需要重新设计权利链，而不是简单购买本项目许可证：

1. 与腾讯签署允许目标产品、账号体系、数据范围、内容呈现、缓存方式和商业模式的书面协议。
2. 获得微信读书名称、图标和其他商标/品牌元素的授权，或使用完全独立且不造成混淆的品牌。
3. 获得所有无明确许可上游的书面授权，或进行干净室替换并保存独立开发证据。
4. 为所有第三方依赖建立软件物料清单和许可证合规包。
5. 禁止把用户缓存、书籍、封面、评论或凭据作为安装包、演示数据或训练数据分发。
6. 建立用户同意、隐私政策、数据最小化、加密、删除、账号隔离、日志脱敏和安全响应流程。
7. 由专业律师审查著作权、合同、个人信息保护、网络安全、消费者保护和设备改装责任。
8. 在所有外部权利解决后，再由本项目权利人决定是否另行签署商业许可。目前没有商业许可承诺。

在这些条件没有全部满足前，商业化的可行方案是停止使用微信读书接口和内容层，只保留经过权利审查的通用墨水屏 UI、排版和设备交互技术，改接已经取得正式授权的内容服务。
