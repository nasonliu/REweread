# 百度 OCR 与 DeepSeek 手写回复配置指南

本文同时面向 AI Agent 和普通用户。Agent 应先读“Agent 操作边界”，再指导用户
亲自在百度控制台和设备浏览器配置页完成敏感操作。

REweread 的 OCR 全部走百度智能云 HTTPS 接口，不要求在手机、电脑或局域网搭建
OCR 服务器。自由手写、荧光笔和符号笔迹不依赖 OCR；普通笔记只有用户主动点击
“识别”时，所选笔迹块才会上传。只有明确选择“AI 回信”笔工具时，停笔后的笔迹块
才会依次发送给百度 OCR 和用户自己配置的 DeepSeek。

## 1. 先分清三项信息

百度 OCR 经典应用凭据通常会显示：

| 字段 | 作用 | REweread 是否填写 |
| --- | --- | --- |
| AppID | 标识百度控制台中的 OCR 应用 | 不填写，只用于用户核对应用 |
| API Key | OAuth `client_id`，用于申请短期 access token | 填写 |
| Secret Key | OAuth `client_secret` | 填写 |

REweread 当前调用经典 AIP OAuth 和手写接口：

```text
POST https://aip.baidubce.com/oauth/2.0/token
POST https://aip.baidubce.com/rest/2.0/ocr/v1/handwriting
```

因此要使用“文字识别 OCR 应用列表”里同时提供 API Key 和 Secret Key 的应用
凭据。不要把只显示一个值的 MCP API Key、AppID、百度账号密码或设备 SSH 密码
填进 API Key/Secret Key 表单。

百度官方快速入门说明 AppID、API Key 和 Secret Key 均可在 AI 服务控制台的
应用列表中获得：
<https://cloud.baidu.com/doc/OCR/s/wkibizyjk>

百度 OCR 文档入口：
<https://cloud.baidu.com/doc/OCR/index.html>

手写文字识别接口与当前能力列表：
<https://ai.baidu.com/ai-doc/index/OCR>

## 2. 用户申请百度 OCR 凭据

百度控制台界面可能调整，以下按 2026 年官方文档整理；名称变化时，以“文字识别
OCR”“应用列表”“API Key”“Secret Key”和“手写文字识别”这些关键词为准。

1. 打开百度智能云并登录自己的账号。
2. 按控制台要求完成实名认证。多数 AI 开放能力的测试额度只向完成认证的账号
   提供；实际免费额度、计费方式和有效期以控制台为准。
3. 进入“产品服务 → 人工智能 → 文字识别 OCR”，打开 OCR 控制台。
4. 进入“应用管理”或“应用列表”，选择“创建应用”。
5. 填写一个容易识别的名称，例如 `REweread-Handwriting`；描述可写“个人设备
   手写笔记识别”。不要在名称或描述中填写任何密钥。
6. 为应用开通“手写文字识别”。如果控制台要求选择接口权限，至少启用文字识别
   OCR 与手写文字识别；若使用自定义权限，确认没有只开通通用印刷体识别。
7. 确认免费测试资源或付费资源可用，并留意配额、QPS 和欠费状态。
8. 创建后，在应用详情或应用列表中找到 AppID、API Key、Secret Key。
9. 只把 AppID 当作应用核对信息；后续设备配置仅需 API Key 与 Secret Key。

百度官方调用说明要求 OCR 图片经 Base64 与 URL 编码后通过 HTTPS POST，请求的
编码后大小不得超过 4 MB；REweread 会在设备内完成缩放、PNG 渲染和编码：
<https://cloud.baidu.com/doc/OCR/s/Ck3h7y2ia>

## 3. 申请 DeepSeek 凭据（仅 AI 回信需要）

AI 回信不是 OCR 的替代品：先由百度把所选笔迹块转成文字，再把这段文字发送给
DeepSeek 生成回复。未配置 DeepSeek 时，百度 OCR 和普通手写笔记仍可单独使用。

1. 在 DeepSeek 开放平台创建自己的 API Key，并自行了解余额、额度和隐私条款。
2. 默认地址是 `https://api.deepseek.com`，默认模型为 `deepseek-chat`；通常无需修改。
3. 不要把 DeepSeek Key 发给 Agent、放进 `.env`、截图或终端命令。只在设备临时
   浏览器配置页填写。

DeepSeek 的 API 使用 OpenAI 兼容的 HTTPS `chat/completions` 接口，并支持流式返回：
<https://api-docs.deepseek.com/>

## 4. 在设备上绑定

准备条件：

- Paper Pro Move 与用于配置的手机或电脑连接同一局域网。
- 设备能够访问 `aip.baidubce.com:443`。
- 设备上已经安装 REweread 2.0.0 或更高版本。
- 用户手边有自己的 API Key 和 Secret Key，但没有把它们发给 Agent。

绑定步骤：

1. 打开 REweread，进入底部“我的”。
2. 找到“云端识别与 AI 手写回复”，点“开启浏览器配置”。服务只在此时启动，不随设备开机
   常驻。
3. 设备显示一个以 `https://` 开头的局域网地址、6 位配对码和剩余时间。
4. 在同一网络的手机或电脑浏览器中完整输入该地址。必须保留 `https://`，不能
   改成 `http://`。
5. 浏览器会提示临时自签名证书。确认地址与设备屏幕完全一致后，仅为本次局域网
   配置会话选择继续访问。证书、私钥和配对码都不会长期保存。
6. 输入设备显示的 6 位配对码。
7. 在“配置服务”中选择本次要更新的一项：
   - 选择“百度 OCR”时，从百度应用详情复制 API Key 到“百度 API Key”，复制
     Secret Key 到“百度 Secret Key”。AppID 不填。
   - 选择“DeepSeek AI 回复”时，只填写 DeepSeek API Key；地址和模型留空即可
     使用默认值。
8. 点“验证并保存”。设备向百度申请短期 access token；验证成功后页面显示
   “配置完成”。DeepSeek 会发起一个极短的 HTTPS 请求验证 Key、模型和网络。
9. 关闭浏览器页面。成功、取消或十分钟超时都会关闭配置服务并销毁本次配对码。

凭据只保存在设备：

```text
/home/root/.local/share/rm-weread/baidu-ocr.json
/home/root/.local/share/rm-weread/deepseek.json
```

文件以 owner-only `0600` 原子写入。应用和配置页不回显已保存的 Key，也不会把
access token、表单值、模型回复或服务响应体写入日志。每次只覆盖表单中选择的
服务凭据；例如更新 DeepSeek 不会改动或清除百度 OCR 配置。

## 5. 使用 OCR 与 AI 回信

### 魔法笔记本

书架首页的“魔法笔记本”是独立的空白纸页，不依附任何一本书。用笔写下一个问题，停笔约
1.4 秒后只上传这一小段连续笔迹到百度手写 OCR；成功得到文字后，才将文字交给已配置的
DeepSeek。提问笔迹会先淡出并在纸面结算后，回复才以原生笔画逐步写出；下一次停笔会成为
新的问题，不会重传之前写过的内容。右上角“清空纸页”只清除这张临时纸，不影响书内批注或配置。

魔法笔记本可选择霞鹜文楷、马善政、刘建毛草、智勇行、龙藏体和站酷快乐作为回信字形；默认是龙藏体。除字形外，还可选择阿遥、墨页、福尔摩斯、林黛玉、苏轼、居里夫人或爱丽丝等具名人设；每个角色都定义姓名、时代、背景和说话风格。后五种均通过
Google Fonts 的 OFL 字体在构建时下载、校验和安装；不会提交进代码仓库。

### 手写输入法

1. 点搜索框或其他普通文本框，切换到“手写”。密码框只允许英文输入。
2. 在手写板连续写完一组中文。
3. 点“识别”，等待百度返回候选。
4. 候选超过一页时点“上页/下页”，再点正确文字写入输入框。
5. 点“清空”可重新书写。没有点击“识别”时不会上传笔迹。

### 阅读页笔迹块

1. 展开右侧笔胶囊，选择颜色，再单独选择“自由写”。
2. 写完一句或一块相邻文字。原始笔迹先保存在本地。
3. 点该笔迹块，在就地操作条中点“识别”；也可选胶囊里的“识”再点目标块。
4. 应用只把这一块笔迹渲染为 PNG 上传，不识别整页。
5. 识别文字附在该笔迹块旁，原始笔迹不会删除。

纯荧光线、圈、箭头、符号和不想转换的手写内容可以一直只保留笔迹，不需要
OCR。搜索书籍时可直接使用本地拼音，也可用手写 OCR 生成搜索文字。

### AI 手写回复

1. 展开笔胶囊，先任选颜色，再选择“AI”。这会进入自由书写，不会改变普通
   荧光笔和自由笔记的行为。
2. 写完一段相邻文字后停笔约 1.4 秒。应用把它固定成一个独立笔迹块，先经百度
   OCR，再把 OCR 文本发送给 DeepSeek。
3. DeepSeek 每返回一个短句，应用就用霞鹜文楷在原笔迹块旁逐字显现；无需整页
   刷新，也不会遮住原始笔迹。
4. 点笔迹块可选“AI 回复”手动重试；点“删除”会同时删除该块原笔迹、OCR 文本和
   AI 回复。断网、识别失败或模型失败时原笔迹仍保留。
5. 继续书写会自动形成新的笔迹块，不会被并入已经发给 AI 的那一句。

## 6. 常见失败

| 现象 | 先检查什么 |
| --- | --- |
| 配置网页打不开 | 手机与设备是否同网；地址是否完整包含 `https://` 和端口；配置会话是否仍在十分钟内 |
| 浏览器提示不安全 | 本次会话使用临时自签名证书；先核对地址与设备屏幕一致，再选择仅本次继续 |
| 配置未通过：应用凭据或权限 | 是否把 AppID 误填成 API Key；API Key 与 Secret Key 是否来自同一个 OCR 应用；手写文字识别是否已开通 |
| 已配置但识别提示无权限 | 控制台中只取得 token 权限但未开通手写接口；检查接口权限、免费资源、欠费和配额 |
| 无法安全保存 | 不要反复提交密钥；先由 Agent 运行不使用真实凭据的存储自检，检查数据目录与文件权限 |
| 无法连接百度 OCR | 设备 DNS、Wi-Fi、时间、CA 证书或到 `aip.baidubce.com:443` 的连接异常 |
| 没有识别到文字 | 只选中线条/符号、笔迹太小或目标块分组错误；保留原笔迹，重新圈定一块再试 |
| 达到调用限制 | 到百度控制台检查资源包、免费额度、QPS 和计费状态，不要在客户端循环重试 |
| DeepSeek 未接受配置 | 检查 API Key、余额、模型名和 HTTPS 地址；更新 DeepSeek 不会影响百度 OCR |
| AI 写得太早 | 只在“AI”笔工具中才会自动处理；普通“自由写”保持完全手动、纯本地 |

## 7. Agent 操作边界

Agent 必须：

- 先让用户阅读本指南，并指导用户亲自在百度控制台创建应用。
- 让用户只通过设备启动的 HTTPS 浏览器配置页填写百度或 DeepSeek API Key。
- 只检查配置状态、网络结果类别、文件是否存在及权限位；不得读取或输出文件内容。
- 排障时使用假的存储探针或无效网络探针，不得把真实凭据放入命令、环境变量、
  `.env`、聊天、issue、截图、日志或 Git。
- 看到用户已经在聊天中暴露凭据时，建议用户在相应服务控制台轮换或删除，不能把它们
  复制到代码、测试或设备命令中。
- 说明 OCR 是可选功能：未配置百度时，拼音、阅读、荧光笔、自由笔迹和符号笔记
  仍应正常工作。

Agent 不得：

- 要求用户把 API Key、Secret Key、AppID、百度/DeepSeek 账号密码或 SSH 密码发到聊天。
- 创建包含真实凭据的 `.env` 并上传设备。
- 使用 `curl`、命令行参数或调试日志直接测试真实 Key。
- 为了修复配置问题删除 `/home/root/.local/share/rm-weread/`。
- 宣称百度 OCR 是本地识别，或宣称没有点击识别也不会产生任何网络请求之外的
  其他应用网络流量。

## 8. Agent 可执行的无密钥检查

在用户已授权设备诊断后，Agent 可运行应用内的网络与存储自检。自检使用无效凭据
确认百度能返回预期拒绝，或用临时假值验证 owner-only 文件写入后立即清理；不得
输出真实配置文件。

安全检查只报告类别，例如：

```text
ocr_configured=yes|no
credential_mode=0600|unexpected
baidu_https=reachable|unreachable
storage_probe=ok|failed
```

不要打印 `baidu-ocr.json`、HTTP 认证参数、access token、OCR 图片、识别结果、
书名、正文或笔记内容。

## 9. 数据与网络说明

- 本地笔迹先保存在设备；不做 OCR 也可以完整使用手写批注。
- 只有用户点击“识别”时，所选手写板或笔迹块的临时 PNG 才发送给百度。
- 书籍正文、整页截图、完整书架、Cookie 和账号信息不属于 OCR 请求。
- access token 只在内存中短期缓存，不写入持久化配置。
- 百度处理、保留、计费和隐私规则由百度智能云条款与用户账号配置决定，使用前应
  由用户自行审阅。
- AI 回信会把百度 OCR 得到的文字（不是整页截图或原始笔迹）发送给用户配置的
  DeepSeek；DeepSeek 的处理、保留、计费和隐私规则由其服务条款和用户账号配置决定。
